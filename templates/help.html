Quick Start

  1. Open Settings → DNS and add credentials for your DNS provider.
  2. From the Certificates page, enter a domain and click Create New Certificate.
  3. Enable Local Authentication under Settings → Users before exposing CertMate to the internet — it ships with auth off so the first run is friction-free.
  4. API token lives at Settings → API Keys. Auto-generated on first start; click Reveal to copy.

Renewal runs daily and refreshes anything within 30 days of expiry. No cron setup needed.

DNS Providers

CertMate uses the DNS-01 challenge: it places a TXT record on your zone, Let's Encrypt (or your CA) queries DNS, and once the value matches the cert is issued. You need a credential with permission to create+delete TXT records on the target zone.

Configure providers in Settings → DNS. Common ones and the minimum API permission each needs:

Provider Token type Minimum scope
CloudflareAPI token (scoped)Zone · DNS · Edit on the target zone
Route 53IAM access keyroute53:ChangeResourceRecordSets, route53:GetChange on the hosted zone
DigitalOceanPersonal access tokenRead + Write
Azure DNSService principalDNS Zone Contributor on the zone resource
Google CloudService account JSONDNS Administrator (or custom role with dns.changes.create)
PowerDNSHTTP API keyZone write on the configured zone

The dropdown lists 22 providers in total. The four major ones (Cloudflare, Route 53, DigitalOcean, Azure) plus Let's Encrypt-recommended clients ship in the default image. The rest are available via EXTRA_REQUIREMENTS at build time — see the README.

Certificate Authorities

Pick a CA in Settings → CA. You can override per-certificate at create time.

  • Let's Encrypt — Free DV certs, 90 days, no EAB. Rate limit: 50 certs per registered domain per week. Default for most users.
  • DigiCert ACME — Paid DV/OV/EV. Requires External Account Binding (EAB key + HMAC). Get those from your DigiCert account portal and paste them in the CA settings.
  • Private CA — Any ACME-compliant internal CA. Provide the ACME directory URL and, if your CA enforces it, an EAB credential.

Use the Test CA Connection button in Settings to verify reachability and EAB before issuing a real cert — it counts against rate limits otherwise.

API

Every endpoint requires a bearer token from Settings → API Keys: 

Authorization: Bearer YOUR_TOKEN

Common endpoints

  • GET /api/certificates
  • POST /api/certificates/create
  • POST /api/certificates/{domain}/renew
  • GET /{domain}/tls (ZIP bundle)
  • GET /{domain}/tls/{cert|key|chain|fullchain}
  • GET /api/activity
  • GET /api/deploy/history

curl examples

# Create a certificate
curl -X POST http://localhost:8000/api/certificates/create \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"domain":"example.com"}'

# Download full bundle as ZIP
curl -H "Authorization: Bearer $TOKEN" \
  -o example.com.zip \
  http://localhost:8000/example.com/tls

Full reference: Swagger (interactive try-it-out) or ReDoc (read-only).

Multi-account DNS

Useful when you manage zones under different cloud accounts (e.g. one Cloudflare tenant per customer, or separate Route 53 accounts for prod/staging).

  1. In Settings → DNS, click Add Account on the relevant provider. Name it (e.g. production) and paste credentials.
  2. One account can be marked as default — used when a cert create request doesn't pick one explicitly.
  3. When creating a certificate, an account dropdown appears for providers with more than one configured account.
  4. From the API, pass account_id alongside dns_provider.

Backup & Restore

  • What's in a backup: settings.json (DNS credentials, CA config, domains, users) plus every certificate file (.pem, .key, full chain). One ZIP, atomic.
  • Automatic: A snapshot is taken before each configuration change, restore, and before destructive ops.
  • Manual: Settings → Backup → Create Backup. Download for off-host storage.
  • Restore: Upload or pick an existing backup, click Restore. A pre-restore snapshot is taken automatically so you can roll back.
  • Retention: Last 10 by default, configurable.

Troubleshooting

DNS provider authentication failed
Token may be wrong, expired, or scoped too narrowly. Re-check it in your provider's console and confirm it can edit DNS records on the target zone. The Test Connection button on each provider card validates without burning a real ACME attempt.
Certificate creation hangs or times out
Usually slow DNS propagation. Most providers settle within a minute or two, some take up to 10. If a particular provider is reliably slow, raise dns_propagation_seconds in Settings.
Let's Encrypt rate limit hit
50 certs per registered domain per week. The error text from acme is included in the activity log. Switch to the LE staging directory for testing — it has no rate limit and uses a separate trust anchor (browsers won't trust the cert, which is the point).
Deployment status keeps showing "unknown"
CertMate probes the cert at the deployment endpoint(s) you configured. If none are set, status stays unknown — that's expected. Configure deployment targets in Settings → Deploy or click Check All from the dashboard.
UI looks broken / Alpine.js failed to load
A red banner appears at the top of the page when this happens. Usually a strict CSP blocking the Alpine CDN, or an offline container. Try a hard refresh (Cmd+Shift+R / Ctrl+F5) and check the browser console for the blocked URL.

Liveness: /health returns JSON with scheduler, disk, and cert directory checks — useful as an external probe.

Report an issue

When something goes wrong, including the items below makes the fix faster — most reports stall on missing context, not on hard root causes.

  • CertMate version (footer of every page, or /health)
  • How CertMate is running (Docker tag / Kubernetes / bare gunicorn) and host OS
  • Steps to reproduce, with the domain or operation involved if relevant
  • What you expected vs. what happened
  • Browser console errors (F12 → Console) for UI issues
  • Container or gunicorn logs around the failure timestamp (redact tokens before pasting)
  • /health JSON output
  • Screenshots help — especially for layout glitches
