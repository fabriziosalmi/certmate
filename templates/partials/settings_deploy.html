Run shell commands after certificate issuance or renewal

Commands run as the CertMate process user. Only admins can manage hooks.

Available Environment Variables

CERTMATE_DOMAIN, CERTMATE_CERT_PATH, CERTMATE_KEY_PATH, CERTMATE_FULLCHAIN_PATH, CERTMATE_EVENT, CERTMATE_DRY_RUN (only set during Test runs)

Blocked Shell Patterns

For safety, the command field rejects pipes, redirects, command substitution, and chained commands: |, &&, ||, ;, $(...), ${...}, `...`, > /, <<, eval, source. References to sensitive files (.pem, .env, settings.json, private keys, vault tokens) are also blocked. If you need any of those, put the logic in a script file inside the container and call the script path directly (e.g. /opt/scripts/deploy.sh).
